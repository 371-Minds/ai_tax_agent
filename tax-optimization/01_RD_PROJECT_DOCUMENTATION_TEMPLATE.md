Quick Reference: Am I Doing R&D Right Now?
YES, it's R&D if you're:

✅ Solving a problem with no clear solution
✅ Testing multiple approaches to find what works
✅ Building something that doesn't exist yet
✅ Improving existing tech in a novel way
✅ Discovering unexpected problems (and solutions)
✅ Creating those "3am businesses" while solving another problem
✅ Failing at experiments (DOCUMENT THOSE FAILURES!)
NO, it's not R&D if you're:

❌ Following a tutorial step-by-step
❌ Implementing well-known patterns without modification
❌ Routine maintenance or bug fixes
❌ Administrative tasks
❌ Marketing/sales activities
When in doubt: If it involves technical uncertainty and experimentation, LOG IT.
